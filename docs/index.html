Enterprise LND security auditing

lnaudit performs static security analysis for Lightning Network Daemon nodes. It inspects lnd.conf, sensitive file permissions, TLS posture, and network exposure, then outputs a scored report with remediation guidance.

Current release
v0.1.0
Scan mode
Config-only, offline
Security modules
Transport, Keys, Access, Privacy, Channel safety
v0.1.0 supports config-based hardening checks. Live node assessment via gRPC is planned for v0.2.0.

Installation

Use source install for rapid updates, or release binaries for controlled deployments.

Prerequisites

  • Go 1.23 or later
  • Read access to the LND directory on target hosts
  • A running LND process is not required for config-only scans

Install from source

go install github.com/NonsoAmadi10/lnaudit@latest

Download release binary

# Linux (amd64)
curl -LO https://github.com/NonsoAmadi10/lnaudit/releases/latest/download/lnaudit-linux-amd64
chmod +x lnaudit-linux-amd64
sudo mv lnaudit-linux-amd64 /usr/local/bin/lnaudit

# macOS (Apple Silicon)
curl -LO https://github.com/NonsoAmadi10/lnaudit/releases/latest/download/lnaudit-darwin-arm64
chmod +x lnaudit-darwin-arm64
sudo mv lnaudit-darwin-arm64 /usr/local/bin/lnaudit

Build from source

git clone https://github.com/NonsoAmadi10/lnaudit.git
cd lnaudit
make build
./bin/lnaudit version

Usage

Run scans

# Auto-detect LND location
lnaudit scan

# Specify config path
lnaudit scan --config /path/to/lnd.conf

# Specify LND data directory
lnaudit scan --lnddir /path/to/.lnd

Output and filtering

# Table output (default)
lnaudit scan --format table

# JSON output
lnaudit scan --format json

# Show only HIGH and CRITICAL findings
lnaudit scan --min-severity high

# Include INFO findings
lnaudit scan --verbose

# Output only score
lnaudit scan --quiet

CI/CD thresholding

# Fail pipeline on HIGH and CRITICAL
lnaudit scan --fail-on high --format json

# Fail only on CRITICAL (default)
lnaudit scan --fail-on critical

Scoring model

Score starts at 100 and deducts points by severity. Minimum score is 0.

CRITICAL-15 points
HIGH-10 points
MEDIUM-5 points
LOW-2 points
INFO0 points

Rating thresholds

Score rangeRating
90 to 100Hardened
70 to 89Acceptable
40 to 69Needs Hardening
0 to 39Critical Risk
Display filters like --min-severity and --verbose only affect rendered output. Score always uses the full unfiltered report.

Architecture

lnaudit/
├── cmd/                    # CLI commands (scan, version)
├── pkg/
│   ├── scanner/            # Finding model, report, score calculation
│   ├── checks/             # Security check implementations
│   ├── config/             # lnd.conf parser with multi-value support
│   ├── lndpath/            # OS-specific LND path auto-detection
│   └── report/             # Table and JSON output writers
├── docs/                   # Documentation and security research
├── Makefile                # build, test, lint, release
└── .github/workflows/      # CI, release, docs deployment

Development

Tooling

# Install development dependencies
make dev-deps

# Run full quality gate
make check

# Build binary
make build

# Run race tests
make test-race

# Generate coverage report
make coverage

# Cross compile binaries
make release

Adding a new check

  1. Implement check logic in pkg/checks/
  2. Return []scanner.Finding with severity, details, remediation
  3. Wire the check in cmd/scan.go
  4. Add table-driven tests in a matching _test.go file
  5. Run make check before opening a pull request
