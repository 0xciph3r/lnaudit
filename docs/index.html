lnaudit

Security auditing tool for Lightning Network Daemon (LND) nodes. Static analysis of configuration files, file permissions, TLS certificates, and network exposure. No running node required.

$ go install github.com/NonsoAmadi10/lnaudit@latest

Overview

lnaudit performs static security analysis of LND node configurations. It reads your lnd.conf, inspects file permissions on sensitive files (wallet, keys, macaroons), validates TLS certificate properties, and checks for dangerous runtime flags.

The tool produces a scored report with specific remediation steps for each finding. It is designed for node operators, security teams, and CI/CD pipelines.

v0.1.0 supports config-only scanning. Live node scanning via gRPC is planned for v0.2.0.

Installation

Prerequisites

  • Go 1.23 or later (install Go)
  • Access to the LND configuration directory on the target machine
  • A running LND node is not required for config-based scanning

Install from source

go install github.com/NonsoAmadi10/lnaudit@latest

Download release binary

Pre-built binaries are available for Linux and macOS on the Releases page.

# Linux (amd64)
curl -LO https://github.com/NonsoAmadi10/lnaudit/releases/latest/download/lnaudit-linux-amd64
chmod +x lnaudit-linux-amd64
sudo mv lnaudit-linux-amd64 /usr/local/bin/lnaudit

# macOS (Apple Silicon)
curl -LO https://github.com/NonsoAmadi10/lnaudit/releases/latest/download/lnaudit-darwin-arm64
chmod +x lnaudit-darwin-arm64
sudo mv lnaudit-darwin-arm64 /usr/local/bin/lnaudit

Build from source

git clone https://github.com/NonsoAmadi10/lnaudit.git
cd lnaudit
make build
./bin/lnaudit version

Verify installation

$ lnaudit version
lnaudit v0.1.0 (commit: abc1234)

Usage

Basic scan

When run without flags, lnaudit auto-detects the LND configuration directory based on the current operating system and user. 

# Auto-detect LND config location
lnaudit scan

# Specify config path
lnaudit scan --config /path/to/lnd.conf

# Specify LND data directory
lnaudit scan --lnddir /path/to/.lnd

Output formats

# Human-readable table (default)
lnaudit scan --format table

# Machine-readable JSON
lnaudit scan --format json

Filtering

# Show only HIGH and CRITICAL findings
lnaudit scan --min-severity high

# Include informational findings
lnaudit scan --verbose

# Output only the numeric score
lnaudit scan --quiet

CI/CD integration

Use the --fail-on flag to set a threshold. The process exits with code 1 if any finding meets or exceeds the specified severity. 

# Fail the pipeline on any HIGH or CRITICAL finding
lnaudit scan --fail-on high --format json

# Fail only on CRITICAL findings (default)
lnaudit scan --fail-on critical

Path auto-detection

lnaudit checks the following default locations based on your operating system:

PlatformDefault LND directory
Linux~/.lnd
macOS~/Library/Application Support/Lnd (also checks ~/.lnd)
Windows%LOCALAPPDATA%\Lnd

If your lnd.conf specifies a custom lnddir or datadir, the scanner reads that value and adjusts all file paths accordingly.

lnaudit v0.1.0 includes checks across five categories. Each finding includes a severity level, a description of the risk, and a specific remediation command or configuration change.

Scoring Model

lnaudit computes a security score starting at 100 and deducting points for each finding based on its severity. The score has a floor of 0.

SeverityPoint deductionUse case
CRITICAL-15Direct fund loss risk, credential exposure
HIGH-10Significant security weakness
MEDIUM-5Suboptimal configuration
LOW-2Minor hardening opportunity
INFO0Informational, no deduction

Rating thresholds

Score rangeRating
90 to 100Hardened
70 to 89Acceptable
40 to 69Needs Hardening
0 to 39Critical Risk
Display filtering (--min-severity, --verbose) affects which findings are shown in the output. The score is always computed against the full unfiltered report.

Development

Prerequisites

  • Go 1.23 or later
  • golangci-lint (make dev-deps installs it)

Build and test

# Install development tools
make dev-deps

# Run all quality checks (format, vet, lint, test)
make check

# Build binary
make build

# Run tests with race detector
make test-race

# Generate coverage report
make coverage

# Cross-compile release binaries
make release

Adding a new check

  1. Add your check function to the appropriate file in pkg/checks/
  2. Return []scanner.Finding with severity, title, description, and remediation
  3. Wire the check into cmd/scan.go in the runScan function
  4. Write tests in the corresponding _test.go file
  5. Run make check to validate

See CONTRIBUTING.md for full guidelines.
